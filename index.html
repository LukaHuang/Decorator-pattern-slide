複習一下類別圖

Generalization 一般化

說明：物件導向程式中稱為Inheritance-繼承。表示繼承了父類別。

Realization 實現

說明：物件導向程式中的Implementation-實作。表示實作了介面。

aggregation 聚合

說明：弱關連 - 整體(Whole)消失後，部分(part)還是繼續存在。有包含的意思，英文為is-part-of。例如：學生是學校的一部分，學校不見了，學生還是可以到別的學校讀書，並不會因為學校不見跟著消失。

Composition

說明：強關連 - 比聚合更強的包含關係，整體消失，部分也會跟著消失。整體-部分的生命週期是一致的。 如果電話消失了，那麼按鍵也會跟著消失。

Alexandar 觀察了大量的建築

各種建築結構即使屬於同一類型，也會互不相同。

雖然互不相同，但可能都具有很高的品質。

在互不相同的情況下，解決相似問題採取的方式也可能不同。

Alexandar觀察解決相似問題的不同結果，縮小關注範圍，來看清優秀設計的相似之處。

模式

A pattern is a solution to a problem in a context

模式是在一個情境之下，針對一個問題的解法。

設計模式的關鍵特徵

模式的關鍵特徵(1)

Name模式名稱
Intent模式的目的
Problem描述問題本身
Solution解決方案
Consequence使用解決方案後的效果

模式的關鍵特徵(2)

Participants and Collaborators 參與者與協作者(entity)
Implementation 模式的實作方式。ps:實作只是模式的實作方式，並不能算是模式本身。
Generic Structure 一般性結構，顯示模式結構的標準圖。

裝飾者模式的關鍵特徵

Intent 意圖 動態地給物件增加職責
problem 附加功能可能發生於物件的基礎功能之前或之後。
Solution 不使用繼承來擴展一個物件的功能。
Consequence 所增加的功能分別放在Decorator物件中。就可以隨需要增加功能。即使能夠動態的增加功能，物件鏈還是必須源自ConcreteComponent
